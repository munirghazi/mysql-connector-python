                 FILTER ISSUE - IDENTIFIED AND FIXED!

ISSUE REPORTED:
  "When select product list it shows all products and it is not applying 
   the filter"

ROOT CAUSE:
  1. Domain was not applying correctly on form load
  2. Previously selected products were not cleared when filters changed
  3. View configuration was incomplete

FIX APPLIED:
  [OK] Added computed domain field (product_domain)
  [OK] Enhanced onchange to clear selections when filter changes
  [OK] Updated XML views with proper configuration
  [OK] Applied to BOTH wizards (Project and Marketing)

    * Enhanced _onchange_filters() to clear selections
    * Applied to both Project and Marketing wizards

  - views/mysdb_enhanced_views.xml
    * Added invisible product_domain field
    * Enhanced product_ids field with context and options
    * Applied to both wizard views

Changes in mysdb_enhanced_views.xml:
------------------------------------

1. Added invisible tracking field:
   <field name="product_domain" invisible="1"/>

2. Enhanced product field:
   <field name="product_ids" 
          widget="many2many" 
          context="{'tree_view_ref': 'odoo_mysql_connector.view_mysdb_product_tree'}"
          options="{'no_create': True, 'no_open': True}">

Why This Works:
--------------
- Computed field ensures domain is always calculated
- Clearing selection prevents confusion
- View configuration ensures proper widget behavior
- Dual approach (@api.depends + @api.onchange) ensures reliability
