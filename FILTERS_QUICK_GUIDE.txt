 FILTER COMBINATIONS - WHAT YOU'LL SEE
================================================================================

No Filters Applied:
   Result: ALL products in the system
   Use When: You know exactly which products to assign

Store = "Store A" only:
   Result: All products from Store A
   Use When: Working with single store, need all its products

Filter = "Products Without Project" only:
   Result: All unassigned products from all stores
   Use When: Quick cleanup across all stores

Store = "Store A" + Filter = "Products Without Project":
   Result: Only unassigned products from Store A
   Use When: Systematic assignment by store ⭐ RECOMMENDED

Store = "Store A" + Filter = "Without Complete Assignment":
   Result: Store A products missing project OR marketing
   Use When: Data quality cleanup for specific store

================================================================================
 REAL-WORLD EXAMPLES
================================================================================

Example 1: First-time product assignment
=========================================
Scenario: You just imported 500 products from Store A, none are assigned

Steps:
1. Open: Bulk Assign to Project
2. Target: Select "Summer Campaign Project"
3. Filter by Store: "Store A"
4. Filter Products: "Products Without Project" (default)
5. Result: Shows all 500 unassigned Store A products
6. Select: Click checkbox in header to select all
7. Click: "Assign to Project"
8. Done: All 500 products now assigned!


Example 2: Fix incomplete assignments
======================================
Scenario: Some products have project but no marketing account

Steps:
1. Open: Bulk Assign to Marketing
2. Target: Select "Facebook Ads Account"
3. Filter by Store: Leave empty (all stores)
4. Filter Products: "Products Without Complete Assignment"
5. Result: Shows only products missing marketing
6. Select: Choose relevant products
7. Click: "Assign to Marketing"
8. Done: Products now have complete assignments!


Example 3: Reassign products to new project
============================================
Scenario: Moving 50 products from "Old Project" to "New Project"

Steps:
1. Open: Bulk Assign to Project
2. Target: Select "New Project"
3. Filter by Store: "Store B" (project's store)
4. Filter Products: "All Products" (to see currently assigned)
5. Select: Choose the 50 products to move
6. Check: "Replace Existing Project Assignment" ✓
7. Click: "Assign to Project"
8. Result: Updated: 50 (moved to new project)


Example 4: Quick audit and fix
===============================
Scenario: Data audit shows 200 high-priority unassigned products

Steps:
1. Go to: Data Maintenance Audit report
2. Filter: "Unassigned to Project"
3. Sort: By Priority (highest first)
4. Note: Which store these products belong to
5. Open: Bulk Assign to Project
6. Filter by Store: The identified store
7. Filter Products: "Products Without Project"
8. Select: High-value products (check order values)
9. Assign: To appropriate project
10. Verify: Re-run audit report

 TECHNICAL DETAILS (For Developers)
================================================================================

Filter Implementation:
----------------------
• Python: @api.onchange decorator on 'store_id' and 'assignment_filter'
• Effect: Updates domain for product_ids field dynamically
• No server round-trip needed - instant UI update

Code Reference:
• Lines 991-1003: Project wizard filter logic
• Lines 1083-1095: Marketing wizard filter logic

Domain Logic:
-------------
store_id selected:
   domain.append(('store_id', '=', self.store_id.store_code))

assignment_filter = 'unassigned':
   domain.append(('has_project', '=', False))
   OR
   domain.append(('has_marketing', '=', False))

assignment_filter = 'incomplete':
   domain.append(('assignment_status', 'in', ['partial', 'none']))

================================================================================
 TROUBLESHOOTING
================================================================================

Problem: Filters don't update the product list
Solution:
   1. Clear browser cache (Ctrl+F5)
   2. Check module version (must be 17.0.2.0.0+)
   3. Restart Odoo service
   4. Check browser console for JavaScript errors (F12)

Problem: No products showing even though some should be there
Solution:
   1. Try "All Products" filter to see everything
   2. Remove Store filter (set to empty)
   3. Check if products actually exist (go to Products menu)
   4. Run Data Maintenance Audit to verify data
   5. Check product store_id values match expected store

Problem: "Store Mismatch" error when assigning
Solution:
   1. USE the Store filter - select same store as project
   2. Verify project's store: Go to Project form, check Store field
   3. Verify products' store: Go to Product form, check Store field
   4. Only assign products from same store as project

Problem: Already assigned products still showing
Solution:
   1. Use "Products Without Project" filter (not "All Products")
   2. If intentionally reassigning, that's correct behavior
   3. Check "Replace Existing" if you want to update assignments
   4. Check preview count - should show expected number

 BEST PRACTICES

DO:

================================================================================
 QUICK DECISION TREE
================================================================================

Do you know which specific products to assign?
   YES → Use "All Products" filter + manual selection
   NO  → Continue below

Are you assigning to a specific store only?
   YES → Set "Filter by Store" to that store
   NO  → Leave store filter empty

Do products already have assignments you want to keep?
   YES → Use "All Products" or "Products Without..."
   NO  → Use "Products Without..." filter

Do you want to replace existing assignments?
   YES → Check "Replace Existing Project Assignment"
   NO  → Leave unchecked (default)

Ready to assign? Click "Assign to Project/Marketing"!
